Если GitHub Desktop по-прежнему выдаёт "Permission denied" для .vs:

1. Закройте Visual Studio и Cursor (чтобы не блокировали файлы в .vs).

2. В GitHub Desktop убедитесь, что в списке изменений нет папки .vs.
   Если .vs всё ещё показывается к коммиту — снимите галочки с этих файлов и не включайте их в коммит.

3. Чтобы навсегда убрать .vs из репозитория (но оставить папку на диске):
   Откройте терминал (cmd или PowerShell) в корне репозитория и выполните:

   git rm -r --cached "14/.vs"
   или (если репозиторий открыт в папке 14):
   git rm -r --cached ".vs"

   Затем сделайте коммит: "Удаление .vs из репозитория".
   После этого .gitignore будет игнорировать .vs и ошибка не повторится.
